5.1 Мета роботи
Набути практичних навичок розгортання програмної системи аналізу та догляду за шкірою в хмарному середовищі, налаштування бази даних у Docker-контейнері, підключення бекенд-застосунку до віддаленої СУБД, тестування IoT-клієнта у середовищі симуляції WokWi, а також надання доступу до розробленого REST API через Swagger-документацію.

5.2 Хід роботи
Базу даних PostgreSQL було розгорнуто за допомогою контейнеризації у Docker. Для цього було створено та налаштовано Docker-контейнер із СУБД PostgreSQL, після чого цей контейнер було підключено до хмарного сервісу Render як окремий сервіс бази даних. У процесі розгортання були налаштовані змінні середовища (назва бази, користувач, пароль, порт), що використовуються бекендом для підключення. Доступ до розгорнутої бази даних перевірявся через інструменти Render та клієнтські засоби підключення до PostgreSQL.
Адреса доступу до розгорнутої БД PostgreSQL: https://skincare-l0s7.onrender.com/

Рисунок 1 – Розгорнута БД PostgreSQL у Docker-контейнері на Render

Для реалізації та тестування IoT-клієнта було використано віртуальний симулятор мікроконтролерів WokWi. У середовищі WokWi змодельовано схему на базі плати ESP32 з підключеним датчиком DHT22. Було реалізовано програмний код, який ініціалізує сенсор, періодично зчитує значення температури та вологості, формує JSON-повідомлення з полями userId, humidity та temperature і надсилає його HTTP-запитом методом POST на REST-ендпоінт серверної частини системи. У ході симуляції було перевірено коректність з’єднання з віртуальною Wi-Fi мережею, правильність формування запитів та обробку відповідей від бекенда.
Адреса проєкту IoT-клієнта у WokWi: https://wokwi.com/projects/450887248830443521 

Рисунок 2 – IoT-клієнт у середовищі WokWi

Програмну систему (бекенд) було розгорнуто на хмарній платформі Render. Початково проєкт бекенда, що реалізує REST API для роботи з користувачами, станами шкіри, аналізами та курсами лікування, було розміщено у репозиторії на GitHub. Репозиторій підключено до Render, що дало змогу налаштувати автоматичний деплой: при оновленні гілки основної розробки сервіс на Render автоматично збирає застосунок, встановлює залежності та запускає бекенд із підключенням до віддаленої бази даних у Docker-контейнері. Після завершення деплою було перевірено роботу основних ендпоінтів API.
Адреса репозиторію проєкту на GitHub: https://github.com/NureMaksymenkoMarko/skincare 

Рисунок 3 – Розгорнутий Backend-застосунок на Render

Для зручності тестування та демонстрації можливостей серверної частини було налаштовано інтерактивну документацію за допомогою Swagger. На бекенді додано Swagger-конфігурацію, яка автоматично генерує опис доступних REST-ендпоінтів, їхніх методів, параметрів, структур запитів і відповідей у форматі OpenAPI. Swagger UI розгорнуто разом із бекендом і доступний за окремою адресою. Це дозволяє напряму через браузер перевіряти роботу API, відправляючи запити до ендпоінтів без додаткових інструментів на зразок Postman.

Адреса Swagger-документації розгорнутого Backend: https://skincare-l0s7.onrender.com/api-docs

5.3 Висновок
У ході виконання лабораторної роботи було розгорнуто всі основні компоненти програмної системи аналізу та догляду за шкірою в хмарному середовищі. Базу даних PostgreSQL запущено в Docker-контейнері та підключено як сервіс на платформі Render, що забезпечило віддалений доступ до даних для серверної частини. Реалізований IoT-клієнт було успішно протестовано у симуляторі WokWi, де підтверджено коректність зчитування показників датчика та надсилання їх на REST API бекенда. Бекенд-застосунок розгорнуто на Render із використанням репозиторію GitHub, а доступ до реалізованого API забезпечено через Swagger-документацію. Отримані результати демонструють працездатність та інтегрованість усіх компонентів системи в реальних умовах розгортання.
